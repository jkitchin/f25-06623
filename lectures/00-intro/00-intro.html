scipy#

scipy contains numerous libraries for a broad range of scientific computing needs.

Suppose we want to perform the following integral: \(I = \int_0^{4.5} J_{2.5}(x) dx\). The function \(J_{2.5}\) is a special function known as a Bessel function. scipy provides both the integration function, and an implementation of the special function we can use.

This is not code you need to memorize now; it is an example of the kind of code we will learn to write.

from scipy.integrate import quad
from scipy.special import jv

?quad

?jv

To evaluate this integral, we have to define a function for the integrand, and use the quad function to compute the integral. The quad function returns two values, the value of the integral, and an estimate of the maximum error in the integral.

# This is how we define a function. There is a function name, and arguments
# The function returns the output of the jv function.
def integrand(x):
    return jv(2.5, x)


I, err = quad(integrand, 0, 4.5)

I, err

(1.1178179380783253, 7.866317216380692e-09)

“Now we’ve got numbers! From here, we can plot, explore, and experiment. Mistakes? They’re not failures—they’re clues. Errors tell us how the code works.”

Around him floated colorful symbols: numpy arrays, sine and cosine curves, a scipy integral, and matplotlib plots.

“This is the magic of Jupyter,” Pyter continued. “Code cells let you try ideas quickly. Libraries like numpy, scipy, and matplotlib give you powerful tools. And when you practice often, you start to think computationally about problems.”

He grinned. “So remember—don’t just read code. Write it. Run it. Break it. Fix it. That’s how you’ll grow into a real Python problem-solver!”
